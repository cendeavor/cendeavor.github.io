程序员的自我修养

等保2.0 vs 等保1.0

等保2.0与1.0的区别

权威解读
一文读懂 | 等保2.0与等保1.0到底有哪些不同

三个新特点

对象范围扩大

将云计算、移动互联、物联网、工业控制系统等列入标准范围
即，从传统网络和信息系统，向“云移物工大”上扩展
“安全通用要求+新型应用安全扩展要求” -> 要求内容

分类结构统一

“基本要求、设计要求、测评要求”分类框架统一，形成“一个中心、三重防护”结构
即“安全通信网络”、“安全区域边界”、“安全计算环境”在“安全管理中心”支持下三重防护体系架构

强化可信计算、强调几项能力

把可信验证列入各个级别，并逐级提出各个环节的主要可信验证要求
同时，强调提高 安全监测能力、通报预警能力、应急处置能力、态势感知能力 等

五大变化

1. 名称

《信息安全等级保护基本要求》 -> 《网络安全等级保护基本要求》

2. 定级对象

信息系统 -> 网络 + 信息系统 （网络基础设施【广电、电信、专用通信网】、云计算、大数据平台、物联网、工控系统、移动互联系统）

3. 安全要求

“安全要求” -> “安全通用要求 + 安全扩展要求”
第三级安全要求目录
技术要求、管理要求 -> 安全通用要求 + 云计算安全、移动互联安全、物联网安全、工业控制系统安全 扩展要求

4. 控制措施分类结构

技术
物理、网络、主机、应用、数据 安全 -> 安全 物理环境、通信网络、区域边界、计算环境、管理中心。
管理
结构并无变化、名称略有调整

5. 工作内涵变化

定级、备案、安全建设、等级测评、监督检查 五个规定动作 -> 加入 安全检测、通报预警、案事件调查 等措施
